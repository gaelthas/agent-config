# 交付文档模式

标准化交付文档模板，确保项目交付时输出高质量的面向用户文档。

## 何时激活

- 项目交付准备
- 版本发布文档编写
- 运维部署文档更新
- 质量验收报告编写
- 新人接手文档整理

## 安装手册要点

### 必备章节

1. **环境要求** — 硬件、软件、网络（含版本号）
2. **安装步骤** — 分步骤，每步有验证方式
3. **配置说明** — 所有配置项（名称、默认值、说明、示例）
4. **启动验证** — 健康检查端点或验证命令
5. **常见问题** — 至少 5 个常见问题和解决方案
6. **回滚方案** — 完整的回滚步骤

### 最佳实践

- ✅ 每个安装步骤后附验证命令
- ✅ 配置项表格包含"是否必填"列
- ✅ 示例使用实际可运行的命令
- ❌ 不要假设读者熟悉项目
- ❌ 不要省略"显而易见"的步骤

## 使用说明要点

### 必备章节

1. **系统概述** — 一段话说清楚系统做什么
2. **快速入门** — 3-5 步完成核心操作
3. **功能模块** — 每个模块含说明 + 操作步骤 + 注意事项
4. **权限说明** — 角色与权限对应表
5. **FAQ** — 常见问题和解决方案

### 最佳实践

- ✅ 使用"操作→结果"的描述方式
- ✅ 标注容易出错的步骤
- ✅ 提供快捷键和效率技巧
- ❌ 不要使用技术术语，用用户能理解的语言

## 测试报告要点

### 必备章节

1. **测试概述** — 版本、时间、环境、人员
2. **测试范围** — 覆盖模块 + 未覆盖范围及原因
3. **测试结果** — 总体统计 + 缺陷统计
4. **遗留问题** — 未修复的问题及计划
5. **测试结论** — 总体评价和发布建议

### 数据呈现

```markdown
| 指标 | 数值 | 标准 | 状态 |
|------|------|------|------|
| 通过率 | 98.5% | ≥ 95% | ✅ |
| 覆盖率 | 82% | ≥ 80% | ✅ |
| 严重缺陷 | 0 | = 0 | ✅ |
| 遗留问题 | 2 | ≤ 5 | ✅ |
```

## 变更记录要点

### 格式规范

```markdown
## v1.2.0 (2025-03-15)

### ✨ 新增功能
- 用户管理：新增批量导入功能 (#123)
- 报表：新增数据导出为 Excel (#145)

### 🐛 修复问题
- 修复登录页面在移动端显示异常 (#156)

### ⚡ 优化改进
- 列表查询性能优化，响应时间降低 60%

### ⚠️ 破坏性变更
- 用户 API 返回格式调整，需更新前端 v1.2+
- 数据库需执行迁移脚本 migration_v1.2.0.sql

### 📋 升级步骤
1. 备份数据库
2. 执行迁移脚本
3. 更新后端服务
4. 更新前端版本
5. 验证核心功能
```

## 交付清单模板

```markdown
# 项目交付清单

## 文档交付
- [ ] 安装手册（已验证可独立部署）
- [ ] 使用说明（已经用户评审）
- [ ] 测试报告（数据真实完整）
- [ ] 变更记录（含升级步骤）
- [ ] API 文档（接口已验证）

## 环境交付
- [ ] 生产环境已部署并验证
- [ ] 配置项已填写正确值
- [ ] SSL 证书已配置
- [ ] 监控和告警已设置

## 数据交付
- [ ] 数据库已初始化
- [ ] 初始数据已导入
- [ ] 备份策略已配置

## 培训交付
- [ ] 管理员培训已完成
- [ ] 用户培训已完成
- [ ] 运维交接已完成
```
