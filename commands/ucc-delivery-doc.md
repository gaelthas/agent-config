# Delivery Doc 命令

此命令调用 **delivery-doc-writer** 代理，生成面向交付的标准化文档。

## 命令功能

1. **收集信息** — 从代码、配置、测试结果中提取信息
2. **选择类型** — 根据需要生成对应的交付文档
3. **填充内容** — 将信息填入标准模板
4. **输出文档** — 生成独立的 Markdown 文档

## 支持的文档类型

| 类型 | 说明 | 使用场景 |
|------|------|----------|
| **安装手册** | 环境要求、部署步骤、配置说明 | 项目部署、环境搭建 |
| **使用说明** | 功能介绍、操作步骤、FAQ | 用户培训、功能交付 |
| **测试报告** | 测试范围、结果统计、缺陷分析 | 质量验收、发布评审 |
| **变更记录** | 版本变更、影响范围、升级步骤 | 版本发布、升级通知 |

## 何时使用

以下情况使用 `/ucc-delivery-doc`：
- 项目交付给客户
- 版本发布前准备文档
- 运维部署新版本
- 新成员接手项目
- 质量验收阶段

## 使用示例

```
用户: /ucc-delivery-doc 生成 v1.2.0 的安装手册

用户: /ucc-delivery-doc 根据最近的测试结果生成测试报告

用户: /ucc-delivery-doc 生成用户管理模块的使用说明

用户: /ucc-delivery-doc 整理 v1.1.0 到 v1.2.0 的变更记录
```

## 输出位置

文档将生成到 `docs/delivery/` 目录：

```
docs/delivery/
├── [项目名]-安装手册.md
├── [项目名]-使用说明.md
├── [项目名]-测试报告-v1.0.0.md
├── [项目名]-变更记录.md
└── [项目名]-API使用指南.md
```

## 与其他命令集成

生成文档前：
- 使用 `/ucc-verify` 确保所有测试通过
- 使用 `/ucc-test-coverage` 获取覆盖率数据
- 使用 `/ucc-quality-gate` 进行质量门禁检查
