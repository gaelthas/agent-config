# Design Doc 命令

此命令调用 **design-doc-writer** 代理，生成标准化的设计文档。

## 命令功能

1. **需求分析** — 理解功能需求和业务背景
2. **选择模板** — 根据需要生成对应类型的文档
3. **填充内容** — 从代码和需求中提取信息
4. **输出文档** — 生成 Markdown 格式的设计文档

## 支持的文档类型

| 类型 | 说明 | 使用场景 |
|------|------|----------|
| **PRD** | 产品需求文档 | 新功能启动、需求评审 |
| **技术方案** | 技术实现方案 | 技术评审、开发指导 |
| **架构设计** | 系统架构文档 | 架构评审、系统级设计 |
| **接口文档** | API 接口定义 | 前后端对接、API 发布 |
| **数据模型** | 数据库设计文档 | 数据库设计评审 |

## 何时使用

以下情况使用 `/ucc-design-doc`：
- 启动新功能开发
- 重大架构变更前
- 技术评审准备
- 前后端接口约定
- 数据库设计评审

## 使用示例

```
用户: /ucc-design-doc 为用户通知系统生成技术方案

用户: /ucc-design-doc 生成订单模块的接口文档

用户: /ucc-design-doc PRD 新增的积分功能
```

## 输出位置

文档将生成到 `docs/` 目录：

```
docs/
├── prd/
│   └── [功能名]-prd.md
├── tech-design/
│   └── [功能名]-技术方案.md
├── architecture/
│   └── [系统名]-架构设计.md
├── api/
│   └── [模块名]-接口文档.md
└── data-model/
    └── [模块名]-数据模型.md
```

## 与其他命令集成

设计完成后：
- 使用 `/ucc-plan` 将技术方案分解为实施计划
- 使用 `/ucc-tdd` 按方案开始测试驱动开发
- 使用 `/ucc-db-review` 审查数据模型设计
