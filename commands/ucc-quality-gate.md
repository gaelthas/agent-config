# Quality Gate 命令

此命令执行提交前的全面质量门禁检查，确保代码符合团队的质量标准。

## 命令功能

1. **代码审查** — 调用 code-reviewer 或语言专用审查代理
2. **测试检查** — 验证测试通过和覆盖率
3. **安全扫描** — 调用 security-reviewer 检查漏洞
4. **文档检查** — 确认文档是否与代码同步
5. **构建验证** — 确保项目可以成功构建

## 何时使用

以下情况使用 `/ucc-quality-gate`：
- 提交 PR 前
- 版本发布前
- 合并到主分支前
- 代码审查会议前

## 检查清单

### 阶段 1：代码质量
- [ ] 代码符合编码规范（函数 < 50 行，文件 < 800 行）
- [ ] 无深层嵌套（> 4 层）
- [ ] 命名清晰，无硬编码值
- [ ] 正确的错误处理

### 阶段 2：测试
- [ ] 所有测试通过
- [ ] 测试覆盖率 >= 80%（或团队约定值）
- [ ] 新增功能有对应测试

### 阶段 3：安全
- [ ] 无硬编码密钥（API 密钥、密码、令牌）
- [ ] 用户输入已验证
- [ ] SQL 注入防护（参数化查询）
- [ ] XSS 防护

### 阶段 4：文档
- [ ] API 变更已更新接口文档
- [ ] README 已更新（如有必要）
- [ ] 变更记录已更新

### 阶段 5：构建
- [ ] 项目可以成功构建
- [ ] 无编译警告（或已记录）
- [ ] 依赖项无已知漏洞

## 输出格式

```markdown
# 质量门禁报告

## 检查结果

| 阶段 | 状态 | 详情 |
|------|------|------|
| 代码质量 | ✅ 通过 | 无关键问题 |
| 测试 | ✅ 通过 | 覆盖率 85% |
| 安全 | ⚠️ 警告 | 1 个低风险项 |
| 文档 | ✅ 通过 | 文档已同步 |
| 构建 | ✅ 通过 | 构建成功 |

## 总体评估：✅ 可以提交

## 待处理项
1. [低风险] 安全警告详情
```

## 门禁规则

| 问题类型 | 严重 | 警告 | 建议 |
|----------|------|------|------|
| 安全漏洞 | 🚫 阻断 | ⚠️ 需处理 | ℹ️ 记录 |
| 测试失败 | 🚫 阻断 | — | — |
| 覆盖率不足 | ⚠️ 需处理 | ℹ️ 记录 | — |
| 代码规范 | ⚠️ 需处理 | ℹ️ 记录 | ℹ️ 记录 |
| 文档缺失 | ⚠️ 需处理 | ℹ️ 记录 | — |

## 与其他命令集成

质量门禁不通过时：
- 代码问题 → `/ucc-code-review` 详细审查
- 测试问题 → `/ucc-tdd` 补充测试
- 安全问题 → 使用 security-reviewer 代理
- 构建问题 → `/ucc-build-fix` 修复构建
